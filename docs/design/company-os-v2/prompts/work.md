# Work · V2 expected prompt

Use case: ui-mockup. Asset: shippable desktop web-app product mockup, 1536×1024.

Create a world-class company Work command centre centred on Milestones and
typed WorkItems. There is no Project object and no Task Graph. The page should
make all formal work visible across Humans and Standing Agents while remaining
calm, elegant, and highly scannable.

Shared Company OS shell with Work active. Header: “Work”, description “Company
outcomes, responsibility, and delivery”, coral “New WorkItem” action. Directly
below, a compact local tab row: Overview, Milestones, All Work, My Work, Agent
Work, Human Actions, Workload. Use a refined filter/search strip with type,
owner, state, and module filters.

First major surface is a horizontal Milestone roadmap, not cards: three
connected milestone bands with target dates and fine progress bars.
“Application package ready” is at risk with 2 of 4 WorkItems complete;
“July content calendar” is on track; “Document architecture baseline” is
achieved. Selecting the first Milestone drives the ledger below.

Second major surface is a dense but beautiful WorkItem ledger with grouped rows
and light column rhythm: State, WorkItem, Type, Accountable, Assignee, Due,
Execution, Approval. Include realistic rows: “Confirm trademark classes” legal
completed; “Run similarity research” research in review with Mission drill-in;
“Prepare filing application” legal in progress assigned to Trademark Agent;
“Approve filing commitment” finance waiting for Brand Owner Human; “Implement
document relation view” development blocked with Git Issue and Pull Request
references. Use typed icons and avatars, not colorful pills everywhere. Right
Context Rail shows Milestone outcome, acceptance criteria, remaining work,
responsibility distribution, one Human decision, and “Open milestone”.

Visual language: warm ivory application, milk-white work surface, graphite
text, restrained coral focus, sage/amber/blue semantics, hairline borders,
compact row density, premium typography, subtle depth. Constraints: no Project,
no dependency graph, no Kanban-only layout, no Wave presented as Milestone, no
generic analytics cards, no purple gradient, no watermark, no browser chrome.
