## What materially improved from V1

- Home is an editorial operating brief rather than a sparse card dashboard.
- Docs behaves as a nested authored knowledge workspace with embedded live
  company views.
- Organization communicates Human Owner -> Lead Agent -> direct Standing
  Agents and keeps External and Proposed identities outside active hierarchy.
- Lead Agent has a true long-lived collaboration workspace and folds one-time
  Mission/Team execution into durable business summaries.
- Business Module demonstrates custom composition over shared company truth.
- Work expresses Milestones and typed WorkItems without Project or Task Graph.
- five durable Agents now have a coherent, role-specific avatar family.

## Known direction-level revisions before approval

- Generated in-image copy and dates are illustrative. Canonical implementation
  copy must come from the fixture and page contracts, not OCR of these images.
- Shared navigation labels vary slightly between generated pages; the approved
  shell must use `Home / Docs / Organization / Work / Approvals / Finance` and
  the separate `Missions / Workflows / Agent Teams` Execution group.
- The new avatar assets are presented separately in this pass. A subsequent
  image revision should place the selected avatars into Organization, Lead
  Agent, Business Module, Home, Docs, and Work without changing actor type or
  authority labels.
- Work's implementation contract must never show a pending filing commitment
  as an already satisfied acceptance criterion. The generated image is layout
  intent only; state truth remains the page contract.
- A proposed BusinessModule cannot expose active mutation actions until its
  activation policy allows them. Review should choose either an Active
  operating page or a Proposed read-only preview before implementation.
