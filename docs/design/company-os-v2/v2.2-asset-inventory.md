# Company OS V2.2 asset inventory

This inventory converts the generated V2.2 visual direction into production
asset responsibilities. Generated mockup pixels are references; they are not
assets to crop and ship.

## 1. Product identity

| Asset | Form | Required variants | Primary placements |
| --- | --- | --- | --- |
| Company OS mark | vector mark | 20, 28, 40 px; monochrome and coral | shell, sign-in, exports |
| Company wordmark | live text + mark | full and compact | desktop shell, mobile header |
| subtle paper texture | repeatable raster or CSS texture | light only; reduced-texture mode | canvas background |

## 2. Actor identity

The canonical portrait sources already live under `identity/avatars/`.
Implementation derives controlled crops rather than creating new identities:

- 24 px: simplified face crop, no decorative orbit;
- 32 px: face crop with actor-type ring;
- 40–48 px: role portrait for lists and activity;
- 64–96 px: focus card or organization node;
- 128 px and above: Agent workspace header or editorial briefing.

Actor type remains visible through shape/ring and label:

- Human: warm sand ring;
- Standing Agent: sage ring;
- External: muted blue ring;
- temporary MemberRun: graphite double ring plus execution badge;
- Proposed actor: coral dashed ring and governance marker.

## 3. Product-object icon family

Implement one outline icon family with a shared optical box and stroke weight.
Use a maintained icon library as the base; do not redraw generated screenshot
glyphs.

Required objects:

- Document, BusinessModule, TypedRecord, Relation, View;
- Milestone, WorkItem, Assignment, Approval;
- Commitment, Payment, MetricObservation, Evidence;
- Human, Standing Agent, External, Service;
- Mission, Wave, Agent Team, Dynamic Workflow, Host execution;
- Conversation, Message, Activity, Handoff, Artifact, Presence, Proposal.

## 4. WorkItem type glyphs

- development;
- design;
- research;
- content;
- legal;
- procurement;
- finance;
- operations;
- governance;
- Human Action;
- general.

Type glyphs communicate classification only. State is always a separate visual
channel so, for example, a legal WorkItem can independently be blocked,
waiting, in review, or completed.

## 5. BusinessModule emblems

P0 production emblems:

- Trademark Management — registration seal + document mark;
- Document Architecture — linked document lattice;
- Finance — ledger line + controlled value mark;
- Content Operations — editorial rhythm + distribution path.

Each emblem needs:

- 24 px compact tree/list form;
- 40 px relationship-card form;
- 64–80 px module header form;
- single-colour fallback;
- no embedded text and no state colour baked into the drawing.

## 6. Page art motifs

| Motif | Implementation | Rule |
| --- | --- | --- |
| company pulse line | CSS/SVG path | decorative; never a metric chart |
| knowledge constellation | CSS/SVG line field | header negative space only |
| organization constellation | CSS/SVG reporting field | behind nodes, non-interactive |
| coordination orbit | portrait frame/SVG | Lead identity only |
| filing lifecycle | live object icons + SVG path | reflects real lifecycle state |
| Milestone route | live milestones + SVG path | route order follows data |

## 7. Interaction and accessibility

- icons always have text or accessible labels when they perform actions;
- decorative art is hidden from the accessibility tree;
- role and state never rely on colour alone;
- reduced motion replaces drawing/pulsing with stable highlighted endpoints;
- high-contrast mode removes paper texture and preserves object boundaries;
- missing portraits fall back to deterministic actor initials and actor-type
  rings, never a random generated face.
