# Organization · V2 expected prompt

Use case: ui-mockup. Asset: shippable desktop web-app product mockup, 1536×1024.

Create a beautiful, calm Organization page for an AI-native company. The page
must immediately explain Human Owner -> Lead Agent -> direct second-level
Standing Agents, their authority, work pressure, and a missing capability. It
is a structural canvas, not a people-card grid or online runtime roster.

Use the shared Company OS shell with Organization active. Main header:
“Organization”, description “Responsibility, authority, and capability”, view
switch “Structure / Workload / Changes”, and coral “Propose agent” action.
Central canvas occupies most of the page with generous warm whitespace and
fine curved reporting lines. At the top, a compact Human Owner node “Brand
Owner · Human” with authority mark. Below, a larger highlighted “Company Lead ·
Lead Agent” node showing 3 active Milestones, 1 blocker, and a tiny availability
indicator. Below it, four aligned direct-report nodes: Document Architecture
Agent, Finance Agent, Content Strategy Agent, and a dashed proposed Trademark
Agent. Each node shows explicit Standing Agent type, role, one concise
responsibility, and WorkItem load. The proposed Agent node visually signals a
governed change, not an active member. External Lawyer appears as a small
bounded collaborator linked to Trademark work, not inside reporting hierarchy.

Right Context Rail for Company Lead: reports to Brand Owner; can assign
WorkItems and start one-time Mission/Team/Workflow execution; cannot approve
financial/legal/organization-wide authority changes; direct reports; one open
capability proposal; button “Open Lead workspace”. Include a small explanatory
legend for Human, Standing Agent, External, Proposed.

Visual language: premium editorial product UI, warm ivory, milk white, graphite,
restrained coral, sage/amber semantics, hairline borders, minimal controlled
shadows, crisp typography. Constraints: no AgentTeamRun members, no provider or
model labels, no runtime cards, no deep department hierarchy, no Project, no
purple gradient, no watermark, no browser chrome.
