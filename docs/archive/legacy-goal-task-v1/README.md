# Legacy Goal / Task Graph v1 archive

This is historical interpretation material for the retired `Goal`, `GoalPhase`,
and Task Graph model. It is not active product, execution, or UI authority.

The active product starts at [Company OS](../../company-os/README.md). The
native execution hierarchy is [Mission/Wave](../../decisions/0026-mission-wave-architecture.md),
and the retirement decision and remaining removal work are recorded in
[ADR 0028](decisions/0028-retire-goal-phase-task-graph.md) and the
[legacy removal plan](company-os/legacy-removal-plan.md).

This documentation archive contains only historical design and ADR provenance;
it is distinct from the required, verified per-project runtime-ledger export.
Do not add it to the active documentation registry, load it into default Agent
context, or use it to design new Company OS, Mission/Wave, WorkItem, or
custom-page behavior.

## Contents

- `VISION.md`: former Goal/GoalPhase closed-loop product vision.
- `goal-learning-loop.md` and `goal-phase-loop.md`: former planning, learning,
  and runtime model.
- `dashboard/`: former Vision/Goal/Task work-board and Goal Workbench specs.
- `decisions/`: ADRs 0006, 0009, 0019, and 0024, retained solely to interpret
  old records and historical implementation choices.
- `decisions/0028-retire-goal-phase-task-graph.md` and
  `company-os/legacy-removal-plan.md`: the completed retirement decision and
  its historical export/deletion plan.
- `core-modules.md`, `agent-control-plane.md`, and `issues/`: former product,
  control-plane, and execution-mode framing.
- `dashboard/design-principles.md`, `dashboard/read-model.md`,
  `dashboard/acceptance.md`, and `dashboard/layout-history.md`: former
  Workbench contracts and decision history.
- `integration/codex-source-audit.md` and `design/internal-workflows/`:
  historical provider audit and internal-workflow source material.
