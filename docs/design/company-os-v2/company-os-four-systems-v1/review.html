Docs × Organization × Work × Finance

四个模块不是四份重复数据，而是四种互相链接的公司真相。阅读下面的图时：盒子表示“谁拥有”，箭头表示“谁请求、谁引用、结果回到哪里”。

谁管理谁，谁只是协作

组织规则：业务 Agent 不直接挂在 Lead、Docs、Work 或 Finance Governance Agent 下面。Work 可以路由任务，Docs 可以要求结果回写，Finance 可以执行财务控制，但新增、权限、组织位置、评估和退休始终由 Org/HR 治理。

各自分管什么

Docs · 上下文与记忆知识、业务结构、来源、决策和最终结果。
Organization · 身份与权力谁存在、向谁负责、能做什么、谁有权批准。
Work · 承诺与责任要做什么、谁负责、处于什么状态、怎样执行和验收。
Finance · 金钱与实际影响预算、承诺、发票、支付、退款和财务指标。

它们怎样完成一次真实工作

最重要的产品规则：Approval 是敏感动作的治理桥梁；它不是普通评论，也不是 Wave gate。Mission/Wave、Agent Team 和 Workflow 只是 Work 的执行引用，不会吸收公司责任、权限或财务真相。
