# Live PRD OS-architecture Expected prompt

## Reference roles

- Four-system responsibility map and governance-led Organization:
  **architecture content and style references**.
- Approved Live PRD homepage: **report style reference**.
- Product Journey Explorer: **page and object handoff reference**.

## Prompt contract

Design an exact `1536×1024` editorial architecture plate titled “Company OS
product architecture” with the subtitle “Pages compose the company. Systems
own the truth.” Explain five layers:

1. Product experience: Home, Docs, Organization, Work, Approvals, Finance and
   their focus pages.
2. Business lines and modules: cross-system lenses, never separate stores.
3. Governed product services: Query/Projection, View composition, Relation
   resolution, Governed Action, Human Approval bridge, Audit/evidence.
4. Four systems of truth: Docs, Organization, Work and Finance with explicit
   object ownership.
5. Durable substrate: append-only Stores, latest projections, schemas/rules,
   Action audit, canonical IDs and provenance.

The right rail shows the governance model, invariants and honest partial state.
Execution receives only a thin bottom boundary: `ExecutionRef` is linked
evidence; its internals are intentionally out of scope. Do not explain
Mission/Wave, Teams, Workflows, Host, providers or plugins.
