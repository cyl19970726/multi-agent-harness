# Live PRD content and asset map

## Report narrative

1. **Product mission** — Docs and Organization are the two cores; Work and
   Finance are first-class operating systems; execution is the shared substrate.
2. **Core pages and handoffs** — Home, Docs, Organization, Work, Approvals and
   Finance cooperate over linked records; Missions, Workflows and Agent Teams
   are drill-ins for how selected work ran.
3. **Four systems of truth** — each system owns one class of truth and other
   pages render linked projections.
4. **Product journeys and business lines** — actual page types and their
   navigable handoffs across Company, Brand & IP, Content, and Development.
5. **Governance-led Organization** — Human Owner, Lead Agent, four Governance
   Agents, and Business Agents beneath Org/HR.
6. **Trademark example** — Document to WorkItem, Actors, ¥3,000 Commitment,
   Human Approval, execution evidence, completion, and result writeback.
7. **Implementation truth** — contract-to-schema-to-Store-to-Action-to-UI-to-
   acceptance status for Docs, Organization, Work, and Finance.

Execution appears only as a lower boundary and linked `ExecutionRef` in this
version. Mission/Wave, Agent Team, Workflow, and Host internals are not expanded.

## Truth labels

Every visual receives one visible label:

- **Actual** — browser evidence over an authority-labelled Store projection;
- **Expected** — approved product or architecture intent;
- **Planned** — contract exists but the implementation chain is incomplete;
- **Historical** — useful prior evidence that is not the current target.
