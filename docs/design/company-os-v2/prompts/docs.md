# Docs Workspace · V2 expected prompt

Use case: ui-mockup. Asset: shippable desktop web-app product mockup, 1536×1024.

Create an exceptionally polished AI-native company knowledge workspace. It
should have the clarity and nesting quality of a premium modern document tool,
but Documents, BusinessModules, Milestones, WorkItems, Approval, Finance, and
Standing Agents are native company objects. Warm ivory background, calm
milk-white paper, graphite typography, restrained coral accents, subtle sage,
hairline separators, refined compact controls. No generic dashboard cards.

Shared 240 px Company OS sidebar with Docs active and the standard navigation.
Inside Docs, include a slim 228 px knowledge tree showing Company, Operating
Manual, Brand A, Content Operations, Brand & IP, and “Trademark Management”
with a small Proposed badge. Main canvas is a readable authored home document,
not a file-manager grid. Header: breadcrumb “Company / Operating system”, title
“Company knowledge”, short editorial description, buttons “New page” and “Ask
an agent”.

Compose the document naturally: a quiet callout “How this company works”; an
inline two-column section with “Operating areas” as elegant linked page rows
and “Needs structure” highlighting the proposed Trademark Management module;
a wide embedded live view “Current milestones” with three rows, accountable
actors, status, target, and remaining WorkItems; a compact embedded “Recent
decisions” table; a lower “Agent-maintained spaces” section with Document
Architecture Agent and Content Strategy Agent as typed Standing Agents.
Integrate comments/mentions subtly in the page margin. Right Context Rail is
about the selected Company document: owner Human, maintained by Lead Agent,
relations to 3 modules, 3 milestones, 1 pending approval, and recent activity.

Constraints: no Project object, no detached database application, no tile grid,
no copied finance values, no raw execution transcript, no purple gradient, no
watermark, no browser chrome.
