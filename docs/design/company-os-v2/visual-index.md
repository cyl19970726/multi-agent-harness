## Visual acceptance rule

Every new P0 page follows:

```text
scenario -> baseline -> approved expected -> normalized viewport
-> design specification -> staged implementation -> screenshot iterations
-> overlay/comparison -> product_truth gate + visual_fidelity gate
```

Never infer visual acceptance from a passing build, a generated design, or a
product-truth test. Never overwrite an approved Expected image; create a
versioned design revision and record why.
