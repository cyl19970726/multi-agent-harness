# Trademark native closure visual contract

This workstream binds the native four-system trademark scenario to three
already approved visual targets. It does not generate a new aesthetic direction.

```text
Current Before  = Store-live browser capture at revision 9998f56
Expected        = approved V2.2 Docs and Business Module + Work OS V1 Board
Actual          = browser capture after governed Action closure
Acceptance      = product_truth and visual_fidelity are separate gates
```

The P0 set is intentionally small. Docs proves the source and returned company
memory, Business Module proves the linked four-system operating loop, and Work
Board proves commitments, ownership and lifecycle. Approval and Finance remain
visible within those compositions and retain their focused interaction tests.

Raw browser runs stay under `.visual-evidence/`; this directory retains only
approved intent, specifications, selected comparisons, overlays and reviews.
