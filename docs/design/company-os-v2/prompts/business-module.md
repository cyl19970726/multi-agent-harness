# Business Module · V2 expected prompt

Use case: ui-mockup. Asset: shippable desktop web-app product mockup, 1536×1024.

Create a highly polished custom BusinessModule operating page for “Trademark
Management”. It is an authored domain workspace composed from Documents,
TypedRecords, Relations, Milestones, WorkItems, Approval, Finance, and Actors—not
a generic dashboard and not a data store.

Shared Company OS shell with Docs active and a slim nested knowledge tree where
Brand & IP / Trademark Management is selected. Main header has a small module
glyph, title “Trademark Management”, status “Proposed”, owner Brand Owner ·
Human, maintained by Document Architecture Agent, and primary action “New
application”. Below, an elegant editorial intro describes the operating
process. A slim horizontal lifecycle “Prepare → Review → Approve → File →
Monitor” highlights Approve.

Main two-column authored layout: broad left column begins with “Current
applications”, a sophisticated compact table containing Brand A / CN-2026-018,
Filing preparation, Human approval needed, owner and next deadline. Then an
embedded Milestone section “Application package ready” with four WorkItems,
typed icons for legal/research/finance, clear owners and two remaining states.
Below, “Knowledge & evidence” links filing guide, class research, similarity
report, and lawyer review. Right column begins with an elegant coral-tinted
decision panel “Approve CNY 3,000 filing commitment”, named Human approver,
evidence, and action. Then “Operating team” shows Lead Agent, Trademark Agent,
Finance Agent, External Lawyer with explicit actor types; “Financial truth”
shows one Commitment pending approval and zero Payment without inventing
settlement.

Visual language: refined editorial tool, warm ivory, milk white, graphite,
restrained coral, sage/amber, hairlines, purposeful density and whitespace.
Constraints: no Project layer, no copied finance facts, no generic KPI cards,
no raw execution logs, no automatic active status for proposed Agent, no purple,
no watermark, no browser chrome.
