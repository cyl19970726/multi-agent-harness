# Goal Design

## Goal

Make the self-hosting MVP non-fake by requiring real object protocol,
provider-delivery evidence, review gates, Dashboard visibility, and adapter
surface evidence.

## Scenario Workflow

```text
GoalDesign
  -> task graph contracts
  -> local store and CLI workflow
  -> persistent Codex runtime contract
  -> provider delivery and ingestion
  -> proposal from diff
  -> review gate
  -> Dashboard read model
  -> self-hosting dogfood
  -> Earning Engine adapter pilot
```

## Roles

- Lead: define acceptance and make decisions.
- Provider integration: harden Codex delivery and runtime health.
- Review / Critic: reject fake or incomplete acceptance.
- Dashboard: expose workflow and risk state.
- Adapter / Strategy: prove external project operation remains outside core.

## Evidence Plan

Use command outputs, schema fixtures, provider sessions, failed-delivery
fixtures, review-gate output, Dashboard snapshots, adapter descriptors, and
goal evaluation artifacts.
